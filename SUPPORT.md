# Support

## What This File Is For

This project is an open-source RC/MVP validated core.

`SUPPORT.md` explains how to ask for help, where to report problems, and what level of support to expect from the repository.

## How To Get Help

Use repository issues for:

- reproducible bugs in current behavior
- documentation problems
- usability issues in the current product shell
- scoped feature ideas that respect the current RC/MVP posture

Use `SECURITY.md` for sensitive security reports.

## What To Include

Helpful support requests usually include:

- what you were trying to do
- what command or UI path you used
- what you expected
- what actually happened
- relevant logs, output, or screenshots
- whether the issue affects `ALLOW`, `NEEDS_APPROVAL`, or `DENY` behavior

## Current Support Expectations

SafeCore is not a managed service and does not offer an SLA.

The repository currently supports:

- local evaluation
- demo and reference product flows
- contributor discussion through issues and pull requests

It does not currently promise:

- production support
- enterprise onboarding
- custom integrations on demand
- real-time incident handling

## Scope Reminder

Please keep requests aligned with the current project posture:

- open-source
- RC/MVP
- validated core
- not a production-ready platform
