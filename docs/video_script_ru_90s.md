# Сценарий Видео На 90 Секунд (RU)

AI-агенты могут очень быстро переходить от вывода модели к вызову инструмента.

Это удобно.

Но без контроля это превращается в риск.

SafeCore — это слой, который стоит между агентом и инструментами, к которым он хочет обратиться.

Он оценивает запрос до выполнения.

Применяет policy.

Проверяет data и tool safety.

Может потребовать approval.

Сохраняет audit evidence.

И при этом удерживает текущее выполнение в dry-run-only режиме.

В основе SafeCore три состояния решения.

`ALLOW` означает, что низкорисковое действие может пройти через guarded path.

`NEEDS_APPROVAL` означает, что действие остаётся заблокированным, пока нет явного approval.

`DENY` означает, что действие слишком опасно и не должно выполняться.

Почему это важно?

Потому что главная проблема agent-систем — не только в планировании.

Главная проблема в контроле.

Кто решает, можно ли реально выполнять действие?

Где находится approval?

Где остаётся audit trail?

SafeCore нужен именно для того, чтобы дать на это понятный ответ.

Сейчас проект нужно оценивать как open-source RC/MVP validated core.

У него уже есть runnable demo path, guarded execution model и первый практический connector path:

allowlisted read-only HTTP status flow для доверенных локальных endpoint'ов.

Это не production-ready platform, и проект этого не обещает.

Если вы хотите быстро понять его ценность, сначала посмотрите на три исхода:

allow,

needs approval,

и deny.

А затем откройте UI и запустите first practical use case, чтобы увидеть, как SafeCore уже сегодня может стоять перед реальной connector boundary.

В этом и есть SafeCore за полторы минуты.
