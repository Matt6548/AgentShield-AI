# Code of Conduct

## Our Standard

SafeCore aims to be a professional, respectful, and technically rigorous open-source project.

Participants are expected to:

- be respectful and constructive
- focus criticism on ideas and implementation, not people
- avoid harassment, abuse, and hostile behavior
- keep discussions useful for maintainers and contributors

## Unacceptable Behavior

- personal attacks
- harassment or intimidation
- discriminatory or abusive language
- doxxing or privacy violations
- repeated bad-faith disruption

## Enforcement

Project maintainers may remove, edit, or reject contributions and discussions that violate this code of conduct.

## Reporting

Use the contact path in `SECURITY.md` for sensitive matters, or repository issues for non-sensitive conduct concerns if a private channel is not needed.
