# Roadmap

## Current Validated Core

SafeCore already provides:

- guarded policy decisions
- approval and deny enforcement
- dry-run execution control
- audit and integrity evidence
- a local product shell for evaluation
- one practical safe integration path

## Near-Term Open-Source Priorities

- keep the public GitHub entrypoint clear and honest
- improve docs, onboarding, and first-run experience
- preserve strong regression coverage for the current safety posture
- deepen narrow practical integration examples without widening scope recklessly

## Intentionally Later

These are important later areas, not promises for the current release:

- production auth/authz
- real connector execution integrations with stronger operational controls
- deployment hardening beyond local/dev baseline
- richer operator workflows and approvals
- broader observability/export integrations
- enterprise-grade governance and support posture
